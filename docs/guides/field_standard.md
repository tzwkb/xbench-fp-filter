# 字段规范 — 案例库与查询集

**制定日期：** 2026/04/27  

---

## 一、核心原则

- **案例来源**：所有 source_text / target_text 来自 xbench 报告，禁止编造
- **术语依据**：以 TB（术语表）为标准，TB 未收录的词条不强制统一
- **标签依据**：每个判断必须对应到具体规则（T1-T4 或 A-G），理由中必须引用规则编号
- **宁写理由，不猜标签**：不确定时写清犹豫点，不交空白理由

---

## 二、判断规则（字段填写参考）

### 2.1 真错误规则（T1-T4）

T 规则优先级高于 A-G。符合任意一条，ground_truth 填「真错误」。

| 规则 | 判定条件 |
|---|---|
| **T1** | 原文字段与 TB 词条**完全匹配**，译文完全未体现该术语 |
| **T2** | 原文是官方机构/行政单位，译文简称导致机构属性丢失 |
| **T3** | TB 有明确拼写规定，译文拼写不一致 |
| **T4** | 原文主要信息在译文中完全消失，无任何替代表达，读者无法获取 |

### 2.2 误报规则（A-G）

仅在**不符合 T1-T4** 时适用。符合任意一条，ground_truth 填「误报」。

| 规则 | 判定条件 |
|---|---|
| **A** | 人物称呼/身份标识被代词或泛指替代，不影响理解 |
| **B** | 地名/场所名在口语中被 here/there/this place 等替代，语义清晰 |
| **C** | AI 检测字段是 TB 词条的子串或超集，错误匹配 |
| **D** | TB 词条适用于 UI/任务场景，但原文是剧情对话 |
| **E** | 译文使用语义等价表达，符合英文习惯，未丢失核心语义 |
| **F** | 译文比 TB 多了 the/of 等语法词，不影响语义 |
| **G** | 地名/场所名是背景信息，译文聚焦事件而省略，故事信息完整 |

## 三、字段规范

### 3.1 source_text（原文）

- **来源**：从 xbench 报告中复制，**不得修改**
- **要求**：完整句子，不截断
- **禁止**：编造、拼接、改写

### 3.2 target_text（译文）

- **来源**：从 xbench 报告中复制，**不得修改**
- **要求**：与 source_text 对应的完整译文
- **禁止**：根据 TB 标准自行修正后填入

### 3.3 error_type（错误类型）

**规则：单值枚举，目前只用“术语违规”。**

| 枚举值 | 适用场景 | MQM 对应 |
|---|---|---|
| 术语违规 | 未按 TB 翻译，漏用规定译名、使用近义词替代 | Terminology |

### 3.4 error_description（错误描述）

**规则：结构化自然语言，60-150 字符，必须包含四要素：原文术语/内容、TB 规定、实际译文、偏差原因。**

**术语违规模板：**
```
术语「{原文术语}」TB 规定为「{标准译名}」，译文使用了「{实际译文}」。
{说明为什么构成违规或为什么属于豁免}
```

**禁止内容：**
- 主观判断：「属于合理处理」、「建议确认」、「仅供参考」
- 上游元数据：「术语ID=22」、「片段编号=48」
- 跨片段引用：「前文第94条也有」
- 冗长背景：「在游戏本地化领域，术语统一是……」
- 重复表达：同一意思说两遍

### 3.5 ground_truth（人工标注标签）

| 值 | 使用条件 |
|---|---|
| 真错误 | 符合 T1-T4 任意一条，或无豁免且译文确实有问题 |
| 误报 | 符合 A-G 任意一条，或 QA 初检本身匹配错误 |
| 争议 | 你与另一位标注者意见不一致，或规则无法明确覆盖 |

`待复核` 是系统内部状态，人工标注时不使用。

### 3.6 reason（复核理由，入库时生成）

该字段不由标注者直接填写，入库时系统根据 ground_truth 和判断规则自动生成。

生成格式要求：不超过 100 字符，必须引用具体规则编号（T1-T4 或 A-G）。

| 场景 | 生成示例 |
|---|---|
| 误报-规则 A | `误报，NPC 对玩家称呼在英文中惯用 you 替代，指代清晰，规则 A` |
| 误报-规则 B | `误报，口语对话中地名被 this place 代替，上下文语义完整，规则 B` |
| 误报-规则 C | `误报，AI 检测字段是术语超集（真气丹≠真气），错误匹配，规则 C` |
| 误报-规则 E | `误报，译文使用语义等价表达，符合英文习惯，规则 E` |
| 真错误-T1 | `真错误，原文与 TB 条目完全匹配，译文完全未体现该术语，T1` |
| 真错误-T2 | `真错误，开封府为官方机构，译文简称丢失机构属性，T2` |
| 真错误-T3 | `真错误，TB 规定 Qinchuan，译文写成 Qin Chuan，拼写不一致，T3` |
| 真错误-T4 | `真错误，该从句具有独立语义（行为动机），译文完全丢弃，T4` |

### 3.7 severity（严重程度）

| 值 | 判断依据 |
|---|---|
| Critical | 主角/关键 NPC 名字错误；核心技能效果误导；主线剧情信息丢失 |
| Major | 重要 NPC 关键对话语义扭曲；关键道具/地名/机构名错误 |
| Minor | 普通术语不一致；非关键术语缺失；不影响理解的拼写变体 |
| Neutral | 冠词差异（a/the）；轻微标点问题；不影响语义的格式差异 |

### 3.8 note（备注，可选）

用于标记特殊处理需求。有值时系统会在 LLM 提示词中注入该备注。

| 使用场景 | 示例 |
|---|---|
| 特殊文本类型 | `剧情对话，人物称呼可省略` |
| 需要特别关注的规则 | `注意：此处为 UI 文本，不适用规则 B` |
| 标注者的犹豫点 | `术语表未收录该词条，按上下文判断` |

**填写要求：** 简洁（<50 字），只写处理指令，不写背景说明。

### 3.9 annotator（标注人）

填写标注者姓名或工号。

  "source_text": "说得好！没想到游侠小小年龄，却颇有见地。",
  "target_text": "Hah! Sharp words. Didn't expect that from someone so young.",

## 五、常见错误与规避

| 常见错误 | 正确做法 |
|---|---|
| 编造 source_text / target_text | 必须从 xbench 报告中复制，不得修改 |
| TB 未收录的词条强行要求统一 | 标注时注明「TB 未收录」，不强制判错 |
| 把「语境意译」判为真错误 | 先检查是否符合规则 B/G，再判 T1-T4 |
| reason 只写「属于误报」 | 必须写明「规则 X」或「T X」 |
| error_description 过短（<30 字） | 补充「原文是什么、TB 规定是什么、实际译文是什么」 |
| error_description 写重复内容 | 同一信息不重复说两遍 |
| 忽略子串匹配问题（规则 C） | 「真气丹」≠「真气」，注意区分完整术语和子串 |
| 把 UI 术语规则套用到剧情对话 | 规则 D：TB 适用于 UI/任务时，剧情对话可能有豁免 |

## 六、入库校验清单

提交前自检：

- [ ] source_text 和 target_text 来自 xbench 报告，未修改
- [ ] 涉及的术语已查 TB
- [ ] error_type 是单值枚举，无分号
- [ ] error_description 60-150 字符，包含原文/规定/实际/原因
- [ ] ground_truth 为 真错误/误报/争议 之一
- [ ] reason 引用具体规则编号（T1-T4 或 A-G）
- [ ] severity 为 Critical / Major / Minor / Neutral 之一
- [ ] ground_truth 为 真错误 / 误报 / 争议 之一
- [ ] 如有 note，内容简洁（<50 字），只写处理指令
