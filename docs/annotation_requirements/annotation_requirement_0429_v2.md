## 标注标准

### 核心原则

**必须阅读原文语境（source_text）和译文语境（target_text），不能只看术语三元组。**

同术语在不同语境下判定可能完全相反：
- "开封府"→"Kaifeng Court"（UI/正式场景）→ **真错误**（机构属性丢失，T2）
- "开封府"→"the authorities"（口语/叙述场景）→ **误报**（代词替代，A）

### 场景分类（先判断）

| 场景 | 特征 | 判定规则 |
|---|---|---|
| **正式场景** | UI界面、系统提示、任务面板、技能/属性描述、公告、角色属性面板 | 术语必须严格统一，**不适用 A-G 豁免** |
| **非正式场景** | 角色口语对话、叙述文本、内心独白、情感描写、剧情背景 | A-G 规则优先适用，T2/T4 例外 |
| **不明确** | 混合或无法判断 | **视为正式场景** |

### 正式场景判定

在正式场景中，术语必须严格使用术语表规定译名。**A-G 规则不适用。**

以下情况判真错误：
- **T1**：术语精确全匹配，译文完全未体现核心语义
- **T3**：专有名词拼写与术语表不一致
- 任何术语缺失/错误/不一致 → 真错误（无豁免）

### 非正式场景判定

在非正式场景中，**先检查 A-G 豁免**，存在则判误报。

**A-G 豁免（判误报）：**
- **A**：人物称谓/身份/势力被代词替代（you/they/someone/our host/the authorities）
- **B**：地名/场所名被 here/there/this place 等替代
- **E**：通用概念同义替换（真气→vital energy，轻击→quick attacks）
- **F**：冠词/介词差异（the/of）
- **G**：叙事焦点转移（背景地名被省略，故事信息完整）

**以下情况即使在非正式场景中也判真错误（无豁免）：**
- **T2**：官方机构/行政单位简称导致属性丢失（如开封府→Kaifeng Court）
- **T4**：核心信息完全消失且无替代（如主语丢失、目的从句被删、核心地名/人名无替代）

**非正式场景复核步骤：**
1. 检查 A-G，触发 → **误报**
2. 未触发，检查 T2/T4，触发 → **真错误**
3. 均未触发 → **误报**

## 输出格式

Excel，每行一条：

| 字段 | 内容 |
|---|---|
| 序号 | 系统自动生成 |
| 类型 | query / case |
| 原文术语 | 术语表中的原文词条 |
| 规定译名 | 术语表中的规定译文 |
| 实际译文 | 当前文本的实际译文 |
| 原文语境 | source_text（完整句子） |
| 译文语境 | target_text（完整句子） |
| 当前标签 | 系统现有标签 |
| **你的判断** | **真错误 / 误报 / 存疑** |
| 理由 | 引用规则编号 + 一句话说明 |
| 场景类型 | 正式 / 非正式 / 不明确 |

**理由示例：**
- "规则A：'大人'在口语中被省略为祈使句，属于代词/泛指替代"
- "规则T2：'开封府'在UI中译为'Kaifeng Court'，机构属性丢失"
- "规则E：'轻击'在叙述中译为'quick attacks'，通用概念同义替换"
