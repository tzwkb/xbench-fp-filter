## 工具改动：xbench.py 新增 --export-queries

**背景：** 此前 xbench.py 只有 `--import`（入库）路径，没有"解析 + 打 ground_truth 标签 + 导出查询 JSON"路径。测试带标注数据集需单独脚本。

**改动：**
- `parse()` 新增 `annotation` 字段：读取 col6/col7 中的 `已修改` 标注（部分行在 col6，部分在 col7，取两列并集）
- CLI 新增 `--export-queries <out.json>`：解析 → 打 ground_truth 标签（`已修改` → 真错误，其余 → 误报）→ 输出 run_test.py 所需 JSON 格式

**用法：**
```
python xbench.py <file.xlsx> --export-queries data/raw_data/<out>.json
python run_test.py real --queries data/raw_data/<out>.json --date <YYYYMMDD>
```

## 第一组：新数据集首跑（5.21a）

**目的：** 在全新数据集（1000 条，100 真错误 + 900 误报）验证昨日 prompt（5.20h + te110 补丁）的泛化能力。

**数据集说明：** 带标注 Xbench 报告，`已修改` = 真错误（共 100 条），无标注 = 误报（共 900 条）。真错误全部来自 Key Term Mismatch。

**结果：** 准确率 74.7%，误报识别率 658/900（73.1%），真错误保护率 89/100（89.0%）。

**漏判的 11 条真错误根因：**

| 触发规则 | 案例 | 根因 |
|---|---|---|
| F①（词序重排） | 醉花阴弟子\|Velvet Shade Disciple → a disciple of the Velvet Shade | LLM 将"Velvet Shade"视为核心名词，"Disciple"视为普通词，误判为词序重排 |
| C②（三者一致） | 九流门弟子\|The Nine Mortal Ways Disciple → a disciple of the Nine Mortal Ways | LLM 认为核心宗派名一致属系统误判，忽视"Disciple"被拆离 |
| F（单复数） | 仙鹤\|Cranes → Crane（技能名）×2 | 技能名中"Cranes"是固定形式，不应以单复数触发 F |
| E（同义替换）×5 | 受击/Hit Stagger、增伤/DMG Boost、跳跃/Jump、幻境/Illusion、调查/Investigate | 游戏操作/战斗/数值术语被当通用词，E 放行过宽 |
| F（格式差异） | 小八\|Xiaoba → Xiao Ba | 拼音连写/分写是专名固定形式，不属于纯语法差异 |
| C②（ground truth 误标） | 冯大哥\|Big Feng → Big Feng | 译文确实使用了"Big Feng"，LLM 判 C② 正确；「已修改」标注来自同一段落的其他改动，非此 QA 项 |

## 第二组：补 E①/F⑤⑥⑦/C② 排除项（5.21b）

**目的：** 针对 5.21a 的 10 条可修漏报（排除 1 条 ground truth 误标），补充排除项。

**改动：**

- **E①** 新增游戏术语具名示例：跳跃/Jump → Leap 错误、受击/Hit Stagger → being staggered 错误、增伤/DMG Boost → increase damage 错误、幻境/Illusion → vision 错误、调查/Investigate → Inspect 错误——均属具名游戏术语，不触发 E
- **F⑤** 宗派/门派+成员身份词构成的固定多词术语（如"Velvet Shade Disciple"），改写为"a [身份词] of [宗派名]"句式属术语结构破坏，不触发 F
- **F⑥** 规定译名中含固定单/复数形式且该词是专有名称组成部分（如技能名中的 Cranes），单复数差异不触发 F
- **F⑦** 专有名词的汉语拼音连写/分写（如 Xiaoba vs Xiao Ba）是名称固定拼写，不触发 F
- **C② 限制** 规定译名为多词短语时（如"The Nine Mortal Ways Disciple"），译文拆分重排（如"a disciple of The Nine Mortal Ways"）三者不完全一致，不触发 C②

**结果：** 真错误保护率 95/100（95.0%，+6pp），误报识别率 623/900（69.2%，-4pp）。规则收紧的副作用——部分原本可放行的误报被压成真错误。

**仍漏判的 5 条：**

| 案例 | 触发规则 | 分析 |
|---|---|---|
| 受击\|Hit Stagger → being staggered when hit | E | LLM 仍认为"受击"在此是动词用法，非独立游戏状态术语 |
| 冯大哥\|Big Feng → Feng（另一条，省略了"Big"） | A/B | 译文省略"Big"，LLM 判指代可接受 |
| 冯大哥\|Big Feng → Big Feng（又一条） | C② | ground truth 误标，LLM 判断正确 |
| 矩天踆乌\|Sun Crow → her enemy | A | 代称，LLM 触发规则 A |
| 赵大哥\|Big Zhao → Brother Zhao | E | "Brother Zhao"被判为合理同义替换 |

其中 1 条（冯大哥 C②）为 ground truth 问题，实际漏判 4 条，有效保护率约 96/99（97.0%）。

## 第三组：补 A 排除项 + E① 补称谓例（5.21c）

**目的：** 针对 5.21b 的 3 条可修漏报（冯大哥→Feng、赵大哥→Brother Zhao、矩天踆乌→her enemy）补规则。

**改动：**
- **A 新增排除①**：术语表中含称谓前缀的固定专名（如「Big Feng」「Big Zhao」），省略或替换称谓前缀（如「Feng」「Brother Zhao」）不触发 A——称谓前缀是规定译名的固定组成部分
- **A 新增排除②**：有官方固定英文名的具名游戏实体（怪物/BOSS/特定构造体等），不得以代词或泛指替代（如「矩天踆乌/Sun Crow」不得替代为「her enemy」）
- **E① 补称谓例**：含称谓前缀的固定专名（如 Big Zhao → Brother Zhao 错误）中的称谓词是规定译名组成，替换不触发 E

**结果：** 准确率 75.8%，误报识别率 660/900（73.3%，+4.1pp），真错误保护率 98/100（98.0%，+3pp）。**两项指标同步提升**，无副作用。

**剩余 2 条漏判：**

| 案例 | 分析 |
|---|---|
| 冯大哥\|Big Feng → Big Feng | Ground truth 误标：译文就是"Big Feng"，LLM 判 C② 正确。有效保护率实为 98/99（99.0%） |
| 漫步\|Stroll → Walk through the woods | E 触发，Walk/Stroll 语义接近，边界案例，暂不修 |
